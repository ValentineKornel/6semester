33. Поясните понятие «TCP-порт»
TCP-порт — это уникальный идентификатор, используемый для связи между приложениями в сети через протокол TCP (Transmission Control Protocol). 
Он является частью адреса, по которому приложение может отправлять и получать данные. Порты разделяются на три категории:

Привилегированные порты (0-1023): зарезервированы для системных служб и приложений (например, порт 80 для HTTP).
Регистируемые порты (1024-49151): могут быть использованы приложениями, которые не имеют фиксированных портов.
Динамические или частные порты (49152-65535): обычно используются для динамического назначения при установке соединений.
Порт помогает различать различные приложения, работающие на одном сервере.

34. Поясните понятие «сетевой сокет»
Сетевой сокет — это программный интерфейс, который позволяет приложениям обмениваться данными по сети. Он реализует 
концепцию «канала» между двумя узлами (например, клиентом и сервером), через который происходит обмен сообщениями. 
Сокет работает через протоколы TCP/IP или UDP и позволяет приложениям устанавливать соединения, передавать и получать данные.

Основные операции с сокетами:

Создание сокета.
Установка соединения (для TCP-соединений).
Отправка и получение данных.
Закрытие соединения.
35. Поясните понятие «WebSocket»
WebSocket — это протокол для двусторонней, постоянной связи между клиентом и сервером по одному сокету. В отличие 
от HTTP, WebSocket позволяет обмениваться данными в обе стороны (от клиента к серверу и наоборот) без необходимости 
повторно устанавливать соединение. WebSocket используется для создания интерактивных веб-приложений, таких как чаты 
или онлайн-игры, где необходима постоянная связь и обмен данными в реальном времени.

36. Поясните процедуру установки соединения между WS-сервером и WS-клиентом
Процедура установки соединения через WebSocket состоит из двух этапов:

HTTP-начало (handshake):
Клиент отправляет запрос на сервер с заголовками, включающими информацию о поддержке WebSocket. Это делается 
с помощью HTTP-запроса типа Upgrade.
Пример HTTP-запроса:
makefile
Copy code
GET /chat HTTP/1.1
Host: example.com
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Key: dGhlIHNhbXBsZSBub25jZQ==
Sec-WebSocket-Version: 13
Ответ сервера:
Если сервер поддерживает WebSocket, он отвечает с кодом 101 (Switching Protocols) и подтверждает установление соединения.
Пример HTTP-ответа:
makefile
Copy code
HTTP/1.1 101 Switching Protocols
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Accept: x3JJHMbDL1EzLkh9t2AR9kFfi6zqzA== 
После завершения handshake устанавливается полноценное соединение WebSocket, и обмен данными может происходить в обе стороны.

37. Поясните понятие «широковещательное сообщение»
Широковещательное сообщение (Broadcast message) — это сообщение, которое отправляется одному источнику, 
но адресуется множеству получателей. В контексте сетевых технологий это может означать отправку данных на 
все устройства в сети или группе пользователей. Например, в мессенджерах или чатах широковещательные 
сообщения могут быть использованы для отправки уведомлений всем участникам чата.

38. Поясните принцип организации потокового ввода/вывода через WS-соединение
Через WebSocket-соединение можно организовать потоковый ввод/вывод (streaming), при котором данные передаются 
непрерывно в обе стороны. Это особенно важно для приложений, которые требуют передачи данных в реальном времени, 
таких как видеоконференции или трансляции данных с сенсоров. Потоковый ввод/вывод работает следующим образом:

Данные отправляются по мере их поступления.
Отправка данных не требует ожидания полного завершения операции, как в случае с обычными HTTP-запросами.
Сервер и клиент могут одновременно отправлять и получать данные в течение всего срока жизни соединения.
39. Поясните принцип действия и назначение механизма «ping/pong»
Механизм ping/pong — это часть протокола WebSocket, используемая для поддержания активного соединения и проверки 
его состояния. Сервер или клиент может отправить сообщение ping, и другая сторона должна ответить сообщением pong. Это служит для:

Проверки работоспособности соединения.
Поддержания соединения активным (особенно в сетях, где могут происходить тайм-ауты из-за простоя).
Обнаружения обрывов соединений.
40. Поясните аббревиатуру «RPC»
RPC (Remote Procedure Call) — это протокол, который позволяет запускать функции или процедуры на удаленном сервере, 
как если бы они выполнялись на локальной машине. При использовании RPC приложение может отправить запрос на выполнение 
функции на сервере и получить результат, не беспокоясь о деталях передачи данных по сети.

41. Поясните принцип работы RPC-механизма, предоставляемого пакетом rpc-websockets
Механизм RPC, предоставляемый пакетом rpc-websockets, позволяет клиентам и серверам обмениваться запросами и ответами 
через WebSocket-соединение. Примерный принцип работы:

Сервер регистрирует доступные функции с помощью метода register.
Клиент вызывает удаленные функции с помощью метода call.
Сервер выполняет запрашиваемую функцию и отправляет результат обратно клиенту.
Кроме того, может использоваться механизм notify, когда сервер отправляет сообщения клиентам без запроса.
42. Поясните принцип действия и назначение механизма «subscriber/publisher»
Механизм subscriber/publisher (публикатор/подписчик) — это модель общения, в которой:

Publisher (публикатор) отправляет сообщения без указания получателей. Он «публикует» информацию.
Subscriber (подписчик) подписывается на получение сообщений определенного типа. Когда публикатор отправляет сообщение, подписчики получают его.
Эта модель используется для реализации однонаправленной связи, например, для распространения новостей или обновлений.

43. Поясните принцип действия и назначение механизма уведомлений
Механизм уведомлений позволяет серверу или клиенту отправлять сообщения другому участнику системы, не требуя от него ответа. 
Уведомления могут использоваться для информирования о событиях или изменениях, которые произошли в системе. Это может быть 
полезно, например, в чатах, системах мониторинга или других приложениях, где важно оповещать участников о происходящих событиях.
